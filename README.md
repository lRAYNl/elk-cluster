Инструкция по развертыванию распределенного комплекса сбора и анализа журналов событий (ELK Stack)

Настоящее руководство описывает алгоритм автоматизированного развертывания отказоустойчивого кластера, включающего компоненты Elasticsearch, Logstash и Kibana, с применением методологии «Инфраструктура как код» (Infrastructure as Code). Разработанная архитектура обеспечивает безопасную передачу данных посредством протоколов SSL/TLS, а также поддерживает балансировку нагрузки между вычислительными узлами.

Раздел 1. Требования к вычислительной среде
Для успешного развертывания комплекса целевые серверы (узлы) должны функционировать под управлением операционной системы семейства Linux. На каждом узле необходимо наличие установленного программного обеспечения Docker и модуля Docker Compose, а также криптографической утилиты OpenSSL для генерации сертификатов безопасности.

Раздел 2. Подготовка конфигурационных параметров
На начальном этапе требуется загрузить файлы развертывания на целевой сервер. Основу конфигурации составляет файл манифеста контейнеров и шаблон переменных окружения. Для адаптации системы под конкретный сервер необходимо создать локальную копию шаблона переменных и указать в ней актуальные параметры сети и аутентификации. К ключевым параметрам относятся: сетевое имя текущего узла, его локальный IP-адрес, список всех IP-адресов, входящих в кластер, а также единые криптостойкие пароли для доступа к базам данных и графическому интерфейсу Kibana. Пароли и ключи шифрования должны быть строго идентичны на всех узлах кластера для обеспечения корректной синхронизации данных.

Раздел 3. Обеспечение криптографической защиты каналов связи
В целях защиты передаваемой телеметрии от агентов сбора к узлам фильтрации (Logstash) требуется сгенерировать уникальные SSL-сертификаты для каждого сервера. Данная процедура выполняется в корневой директории развертывания путем создания специализированной папки для хранения криптографических ключей.

Раздел 4. Инициализация и запуск комплекса
По завершении этапов конфигурации и генерации ключей осуществляется запуск кластера с использованием встроенных команд среды Docker Compose. Система в автоматическом режиме осуществит загрузку необходимых образов контейнеров (включая пользовательский образ Logstash с интегрированными правилами парсинга) и инициирует их работу в фоновом режиме.


Далее идет цикл команд, требуемых для развертки. Заменяйте на свои данные, где это требуется!

-----------------------------------------------------------------
Подготовка файлов и установка зависимостей:
-----------------------------------------------------------------
Для начала, установим нужные компоненты для работы со стеком

``` sudo apt-get update ```

``` sudo apt-get install wget apt-transport-https gnupg2 -y ```

``` sudo apt install docker-compose-v2 ```

Теперь, для работы, нужно установить агента сборщика (если будете использовать не filebeat, то пропускаем). Ссылка на репозиторий:
- https://github.com/lRAYNl/filebeat/

Клонируем репозиторий с конфигурацией

``` sudo git clone https://github.com/lRAYNl/elk-cluster.git ```

Переходим в папку проекта

```cd elk-cluster ```

Открываем и вписываем IP-адрес сервера и пароли

``` sudo nano .env.template ```

``` sudo mv .env.template .env ```

-----------------------------------------------------------------
Настройка безопасности (SSL):
-----------------------------------------------------------------

Создаем директорию для сертификатов

``` sudo mkdir certs ```

Генерируем сертификаты для Logstash (Нужно заменить IP_АДРЕС_УЗЛА на реальный IP)

``` sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout certs/logstash.key -out certs/logstash.crt -subj "/CN=IP_АДРЕС_УЗЛА" ```

ВАЖНО! Генерацию сертификата для Elasticsearch производим один раз на одном сервере, после чего сертификат переносим на все остальные сервера любым способом, где разворачивается elk-стек.

Генерируем "Главную печать" (CA)

``` sudo docker run -u root --rm -v $(pwd)/certs:/certs docker.elastic.co/elasticsearch/elasticsearch:9.2.2 bin/elasticsearch-certutil ca --out /certs/elastic-stack-ca.p12 --pass "" ```

Генерируем кластерный сертификат, подписанный нашим CA

``` sudo docker run -u root --rm -v $(pwd)/certs:/certs docker.elastic.co/elasticsearch/elasticsearch:9.2.2 bin/elasticsearch-certutil cert --ca /certs/elastic-stack-ca.p12 --out /certs/elastic-certificates.p12 --ca-pass "" --pass "" --dns localhost,ИМЕНА_НОДОВ,elasticsearch --ip 127.0.0.1,ВСЕ_IP_НА_КОТОРЫХ_БУДЕТ_ELK ```

Разрешаем чтение всех файлов в папке certs для всех пользователей

``` sudo chmod -R 644 certs/ ```

Делаем саму папку доступной для чтения и выполнения

``` sudo chmod 755 certs/ ```

Нам нужно перекинуть только один файл на все остальные машины. Если машины в локальной сети, можно воспользоваться командой на первом сервере, чтобы раздать файлы содержащиеся в папке. Включаем сервер для передачи сертификата

``` cd certs/ ```

``` sudo python3 -m http.server PORT #в port пишем свой порт ```

На остальных узлах, мы также заходим в папку и пишем эту команду

``` cd certs/ ```

``` sudo wget http://IP:PORT/elastic-certificates.p12 ```

-----------------------------------------------------------------
Настройка агента по сбору логов:
-----------------------------------------------------------------

Для того, чтобы успешно собирать и отправлять логи на logstash, требуется установить filebeat и сбросить конфигурационный файл из репозитория (если используете другой сборщик, можете пропустить этот пункт). Если вы установили filebeat ранее, можете проводить его настройку по тому же репозиторию:

``` https://github.com/lRAYNl/filebeat/ ```

-----------------------------------------------------------------
Запуск кластера:
-----------------------------------------------------------------

Запускаем контейнеры на всех машинах

``` sudo docker compose up -d ```

Проверяем, что все контейнеры успешно запустились и работают

``` sudo docker ps ```

-----------------------------------------------------------------
Настройка .env (пароли) и ILM
-----------------------------------------------------------------

После успешного запуска, нужно создать пароли от сущностей kibana_system и elastic и вставить новые пароли в .env в соответствующие пункты на какждом сервере и перезапустить контейнеры. Переходим на первую машину, и сбрасываем пароли

``` sudo docker exec -it ИМЯ_НОДЫ_ELASTIC bin/elasticsearch-reset-password -u kibana_system -i ```

``` sudo docker exec -it ИМЯ_НОДЫ_ELASTIC bin/elasticsearch-reset-password -u elastic -i ```

Вписываем пароли и дописываем .env

``` sudo nano .env ```

Перезагружаем наши контейнеры

``` sudo docker restart ИМЯ_НОДЫ logstash-ИМЯ_НОДЫ kibana-ИМЯ_НОДЫ ```

Обязательно, в образе logstash существует ILM, в дальнейшем вы сможете удалить или изменить, поэтому нужно после перезапуска (через минуты 3) ввести данные строки:

```
curl -k -X PUT "https://IP:9200/_ilm/policy/filebeat-retention-policy" \
-H 'Content-Type: application/json' \
-u "elastic:ТВОЙ_ПАРОЛЬ" \
-d'
{
  "policy": {
    "phases": {
      "hot": {
        "actions": {
          "rollover": { "max_size": "50gb", "max_age": "30d" }
        }
      },
      "delete": {
        "min_age": "30d",
        "actions": { "delete": {} }
      }
    }
  }
}'
```

И вот это:

```
curl -k -X PUT "https://IP:9200/_index_template/filebeat-custom-ilm" \
-H 'Content-Type: application/json' \
-u "elastic:ТВОЙ_ПАРОЛЬ" \
-d'
{
  "index_patterns": ["filebeat-*"],
  "template": {
    "settings": {
      "index.lifecycle.name": "filebeat-retention-policy"
    }
  }
}'
```

После ввода, снова перезагружаемся:

``` sudo docker restart ИМЯ_НОДЫ logstash-ИМЯ_НОДЫ kibana-ИМЯ_НОДЫ ```

-----------------------------------------------------------------
Переходим для проверки и продолжаем настройку ILM:
-----------------------------------------------------------------

Теперь, можем перейти по внешнему IP адресу, указав порт, требуемого сервера (если разворачивали на несколько серверов ELK), вводите пароль от elastic, и теперь можете перейти во вкладку discover и увидеть логи. Проводить настройку с индексом и настройка ILM проводим в index managment, выбираем созданный индекс, Index Lifecycle Policies, выбираем filebeat-retention-policy, и можем его настроить.

Интеграция с S3:
-----------------------------------------------------------------

Если вам нужна интеграция с s3, то нужно убрать комментарии с трех строчек в docker-compose.yml 
      
  ```
      - s3.client.default.endpoint=${S3_HOST}
      - s3.client.default.protocol=https
      - s3.client.default.path_style_access=true
  ```
Делать это нужно на всех машинах, на которых развернут ELK

Вводим Access Key (консоль попросит вставить ключ и нажать Enter)

``` sudo docker exec -it ИМЯ_НОДЫ bin/elasticsearch-keystore add s3.client.default.access_key ```

Вводим Secret Key (консоль попросит вставить секретный ключ и нажать Enter)

``` sudo docker exec -it ИМЯ_НОДЫ bin/elasticsearch-keystore add s3.client.default.secret_key ```

Дальше нужно делать только на одной, главной машине
Заставляем Эластик применить новые ключи без перезагрузки контейнера (вводим свои данные)

``` curl -X POST "https://IP:9200/_nodes/reload_secure_settings" -u elastic:ТВОЙ_ПАРОЛЬ -k ```

Подключаем s3 хранилище

``` curl -k -X PUT "https://IP:9200/_snapshot/s3-archive" \
-H 'Content-Type: application/json' \
-u "elastic:ТВОЙ_ПАРОЛЬ" \
-d'
{
  "type": "s3",
  "settings": {
    "client": "default",
    "bucket": "ИМЯ_ТВОЕГО_БАКЕТА"
  }
}'
```
